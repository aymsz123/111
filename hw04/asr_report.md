# 开源语音识别(ASR)调研与选型报告
## 一、调研方案概述
本次调研选取3款主流开源ASR方案：OpenAI Whisper、FunASR、Vosk，从许可协议、语言支持、模型性能、部署难度等维度进行全面对比，最终选型OpenAI Whisper完成本地实现。

## 二、三种开源ASR方案详细对比
| 对比维度 | OpenAI Whisper | FunASR | Vosk |
| :--- | :--- | :--- | :--- |
| **许可协议** | MIT License（完全开源，商用免费，无限制） | Apache 2.0（开源商用免费，可修改分发） | Apache 2.0（开源商用免费） |
| **语言与方言支持** | 原生支持99种语言，覆盖中文普通话、粤语等主流方言，多语言能力极强 | 深度优化中文普通话，支持多方言、多语种，中文场景精度行业领先 | 支持20+语言，中文支持基础，方言适配有限 |
| **模型体量与推理速度** | 模型体量从tiny(39MB)到large-v3(10GB)，精度与速度成反比，CPU可流畅运行小模型 | 提供轻量化模型（Paraformer-lite，100MB级），工业级优化，推理速度快，GPU加速效果显著 | 模型体量极小（几十MB），推理速度极快，无GPU也能实时运行 |
| **流式/实时支持** | 原生不支持流式，需第三方插件（whisper-streaming）实现，延迟中等 | 原生支持流式实时识别，延迟低，适配实时对话场景 | 原生支持流式实时识别，延迟极低，适合嵌入式场景 |
| **依赖与部署难度** | 依赖PyTorch、FFmpeg，Python环境一键安装，部署难度极低，开箱即用 | 依赖PyTorch、ONNX Runtime，提供预编译包，部署难度低，中文生态完善 | 提供多语言SDK，无深度学习框架依赖，部署难度极低，跨平台性强 |
| **笔记本/PC实测感受** | 多语言能力强，中文识别精度优秀，小模型可满足实时需求，适合初学者快速上手 | 中文识别精度碾压Whisper，流式体验好，适合中文生产环境，对GPU友好 | 极致轻量化，低配置设备流畅运行，适合嵌入式、低算力场景，精度略低 |
| **仓库来源/版本** | https://github.com/openai/whisper v20240930 | https://github.com/alibaba-damo-academy/FunASR v1.0.0 | https://github.com/alphacep/vosk v0.3.45 |

## 三、选型理由
本次作业最终选择 **OpenAI Whisper** 作为实现方案，核心理由如下：
1.  **开箱即用，适配课程作业**：Python环境一键安装，无需复杂配置，适合大一学生快速完成本地部署与验证。
2.  **多语言通用，兼容性强**：完美适配任务二的中文配音识别，同时支持多语言拓展，满足课程实践的通用性要求。
3.  **社区生态完善，学习成本低**：官方文档详尽，社区问题解决方案丰富，遇到问题可快速定位解决，适合学习与调试。
4.  **模型灵活，适配不同设备**：可根据笔记本算力选择不同大小的模型，兼顾识别精度与运行速度，CPU/GPU环境均可流畅运行。

## 四、实现方案概述
本次实现包含两种实验方式：
1.  **音频文件识别**：使用任务二剪映导出的配音音频，完成离线批量识别。
2.  **麦克风实时输入识别**：基于Whisper实现实时语音流识别，验证方案的实时性。
